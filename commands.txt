remote addКоманда добавляет удаленный репозиторий по указанному URL-адресу. И это позволяет вам назначить псевдоним для этого репозитория, чтобы вам не приходилось вводить URL-адрес при обращении к удаленному репозиторию.

Распространенным именем для удаленного репозитория является origin, но некоторые находят это имя немного запутанным, особенно если удаленный репозиторий не был источником проекта. В этом случае я иногда использую имя локального репозитория с -gh (для GitHub) в качестве псевдонима удаленного репозитория, чтобы я мог легко запомнить его. Но не стесняйтесь использовать любое имя.

Наш локальный репозиторий Git теперь знает, где найти связанный с ним удаленный репозиторий. Следующий шаг - синхронизировать репозитории, чтобы их история фиксаций совпадала.

Извлечь из удаленного репозитория и объединить с нашим локальным репозиторием с помощью команды Git:
git pull recipes-gh master --allow-unrelated-histories

Флаг --allow-unrelated-histories требуется при первом извлечении, потому что мы хотим объединить два репозитория, которые были созданы отдельно и в настоящее время не имеют общей связанной истории.

После первого запуска команде git pull этот флаг не требуется:

git pull recipes-gh master
